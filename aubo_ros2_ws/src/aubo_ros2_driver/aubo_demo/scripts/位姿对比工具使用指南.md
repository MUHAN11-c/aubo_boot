# 位姿对比工具使用指南

## 快速开始

### 1. 准备工作

```bash
# 终端 1: 启动机械臂和 MoveIt2
cd /home/mu/IVG/aubo_ros2_ws
source install/setup.bash
# 启动你的机械臂驱动和 MoveIt2

# 终端 2: 启动机器人状态发布器
cd /home/mu/IVG/aubo_ros2_ws
source install/setup.bash
ros2 run demo_driver robot_status_publisher

# 终端 3: 运行位姿对比工具
cd /home/mu/IVG/aubo_ros2_ws
source install/setup.bash
cd src/aubo_ros2_driver/aubo_demo/scripts
./quick_test.sh
```

### 2. 工具说明

本目录包含以下位姿对比工具：

#### 🔧 quick_test.sh - 快速启动脚本
交互式菜单，选择并启动不同的测试工具

**使用方法：**
```bash
./quick_test.sh
```

#### 📏 compare_pose.py - 手动位姿对比
手动控制机械臂移动，记录并对比移动前后的位姿

**使用方法：**
```bash
python3 compare_pose.py
```

**适用场景：**
- 测试手动操作的精度
- 验证机械臂运动的重复性
- 对比任意两个时刻的位姿

#### 🎯 compare_pose_auto.py - 自动位姿对比
自动调用 MoveIt2 移动机械臂到目标位姿，并对比实际到达位姿

**使用方法：**
```bash
python3 compare_pose_auto.py
```

**适用场景：**
- 测试 MoveIt2 控制精度
- 验证运动规划的准确性
- 评估到位精度

#### 📊 batch_pose_test.py - 批量测试工具
批量测试多个位姿，生成详细的测试报告

**使用方法：**
```bash
# 默认测试 5 个位姿
python3 batch_pose_test.py

# 自定义测试数量
python3 batch_pose_test.py --ros-args -p test_count:=10
```

**适用场景：**
- 系统性能评估
- 重复精度测试
- 生成测试报告

## 工作原理

### 数据来源

1. **目标位姿** - 期望机械臂到达的位置和姿态
2. **实际位姿** - 从 `/demo_robot_status` 主题获取的机械臂当前位姿

### 对比指标

#### 位置差异
- **Δx, Δy, Δz**: 各轴方向的位置偏差（米）
- **总距离**: 空间中的欧几里得距离（通常以毫米显示）

#### 姿态差异
- **欧拉角差异**: Roll, Pitch, Yaw 的偏差（度）
- **四元数旋转角度**: 从目标姿态到实际姿态的最短旋转角度（度）

### 判定标准

- **位置容差**: 默认 1 mm
- **姿态容差**: 默认 0.57°（0.01 弧度）

位姿差异在容差范围内则判定为**匹配 ✅**，否则为**不匹配 ⚠️**

## 参数配置

### 通用参数

所有工具都支持以下参数：

```bash
python3 [工具名].py --ros-args \
  -p position_tolerance:=0.001 \        # 位置容差 (米)
  -p orientation_tolerance:=0.01        # 姿态容差 (弧度)
```

### 自动工具专用参数

```bash
python3 compare_pose_auto.py --ros-args \
  -p velocity_factor:=0.3 \             # 速度因子 (0.0-1.0)
  -p acceleration_factor:=0.3           # 加速度因子 (0.0-1.0)
```

### 批量测试专用参数

```bash
python3 batch_pose_test.py --ros-args \
  -p test_count:=10                     # 测试次数
```

## 常见问题

### Q1: 提示 "/demo_robot_status 话题不存在"

**解决方案：**
```bash
# 检查话题
ros2 topic list | grep robot_status

# 如果话题名不同，需要修改脚本中的话题名
# 或者启动 robot_status_publisher 节点
ros2 run demo_driver robot_status_publisher
```

### Q2: 提示 "/move_to_pose 服务不可用"

**解决方案：**
- 此服务由 demo_driver 包提供
- 确保相关节点正在运行
- 或使用手动对比工具（不需要此服务）

### Q3: 位姿差异总是很大

**可能原因：**
1. 机械臂未标定
2. 坐标系设置不正确
3. 目标位姿超出工作范围

**解决方案：**
1. 重新标定机械臂
2. 检查 base_frame 参数
3. 选择工作空间内的测试位姿

### Q4: 等待机械臂到位超时

**解决方案：**
- 检查机械臂是否真的在运动
- 增加速度和加速度因子
- 修改脚本中的超时时间

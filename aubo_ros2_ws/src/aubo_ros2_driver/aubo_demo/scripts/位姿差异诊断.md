# 位姿差异诊断指南

## 当前问题分析

从测试结果看到，所有位姿都有**系统性的误差**：

### 📊 误差特征

```
测试 #1: 位置差异 26.717 mm (Δz = -26.051 mm)
测试 #2: 位置差异 24.424 mm (Δz = -23.936 mm)
测试 #3: 位置差异 25.193 mm (Δz = -24.228 mm)
测试 #4: 位置差异 22.871 mm (Δz = -22.257 mm)
测试 #5: 位置差异 24.167 mm (Δz = -23.561 mm)

平均 Z 轴误差: -24mm (系统性偏差)
姿态差异: ~2.5-2.8°
```

**关键发现：**
- ✅ **验证1成功**：运动到位判断正常工作
- ⚠️ **验证2失败**：位姿差异大（Z轴约-24mm，姿态约2.6°）
- 📈 **系统性偏差**：所有测试都在Z轴负方向有约24mm的偏移

## 🔍 可能原因

### 1. TCP (工具中心点) 偏移 ⭐⭐⭐ (最可能)

**症状：** Z轴系统性偏移约24mm

**原因：** 
- 手眼标定位姿是相对于**相机坐标系**或**末端法兰**的
- MoveIt2 控制的是相对于配置的 **TCP** 的位姿
- 如果 TCP 配置不正确（例如没有考虑相机或夹具的偏移），就会产生系统性误差

**验证方法：**
```bash
# 检查当前 TCP 配置
ros2 param get /move_group robot_description_kinematics

# 查看 URDF 中的 TCP 定义
# 检查 end_effector_link 到实际工具末端的距离
```

**解决方案：**
1. 检查手眼标定使用的坐标系
2. 确认 MoveIt2 的 end_effector_link 配置
3. 如果有工具/相机，需要在 URDF 中正确定义 TCP 偏移

### 2. 坐标系不一致 ⭐⭐

**症状：** 位置和姿态都有偏差

**原因：**
- JSON 文件中的位姿可能是相对于**相机坐标系**
- `/demo_robot_status` 发布的是相对于 **base_link** 的位姿
- 如果没有正确的坐标变换，两者无法直接对比

**验证方法：**
```bash
# 检查 TF 树
ros2 run tf2_tools view_frames

# 查看相机到 base_link 的变换
ros2 run tf2_ros tf2_echo base_link camera_link
```

**解决方案：**
- 确认 JSON 位姿的坐标系
- 如果是相机坐标系，需要转换到 base_link
- 或者修改验证脚本，在对比前进行坐标变换

### 3. 手眼标定精度问题 ⭐

**症状：** 姿态误差约2.6°

**原因：**
- 手眼标定本身的精度有限
- 标定数据质量不佳
- 标定算法误差

**验证方法：**
- 重新进行手眼标定
- 使用更多标定位姿
- 检查标定残差

### 4. 机械臂标定问题 ⭐

**症状：** 位置和姿态误差

**原因：**
- 机械臂运动学参数不准确
- DH参数偏差
- 关节零点偏移

**解决方案：**
- 重新标定机械臂
- 检查机械臂重复定位精度

## 🔧 解决方案

### 方案 1: 补偿 TCP 偏移 (推荐)

如果确认是 TCP 偏移导致的，可以在验证脚本中补偿：

```python
# 在脚本中添加 TCP 偏移补偿
TCP_OFFSET_Z = 0.024  # 24mm 偏移

def compensate_tcp_offset(pose):
    """补偿 TCP 偏移"""
    # 这里需要根据实际情况计算
    # 简单情况下，如果只是Z轴偏移：
    compensated_pose = Pose()
    compensated_pose.position.x = pose.position.x
    compensated_pose.position.y = pose.position.y
    compensated_pose.position.z = pose.position.z + TCP_OFFSET_Z
    compensated_pose.orientation = pose.orientation
    return compensated_pose

# 在对比前应用补偿
target_pose_compensated = compensate_tcp_offset(target_pose)
```

### 方案 2: 修正 URDF 中的 TCP 定义

编辑机器人 URDF 文件，添加或修正工具/相机的定义：

```xml
<link name="camera_link">
  <!-- 相机或工具的定义 -->
</link>

<joint name="ee_to_camera" type="fixed">
  <parent link="end_effector_link"/>
  <child link="camera_link"/>
  <origin xyz="0 0 0.024" rpy="0 0 0"/>
  <!-- 这里的 0.024m (24mm) 是测量或标定得到的偏移 -->
</joint>
```

### 方案 3: 坐标系转换

如果 JSON 位姿和 /demo_robot_status 使用不同坐标系：

```python
import tf2_ros
import tf2_geometry_msgs

# 创建 TF buffer
tf_buffer = tf2_ros.Buffer()
tf_listener = tf2_ros.TransformListener(tf_buffer)

def transform_pose(pose, from_frame, to_frame):
    """转换位姿到目标坐标系"""
    pose_stamped = PoseStamped()
    pose_stamped.header.frame_id = from_frame
    pose_stamped.pose = pose
    
    transformed = tf_buffer.transform(pose_stamped, to_frame)
    return transformed.pose
```

# GraspNet 数据集使用步骤

已下载原始数据集（zip 文件）后，按以下顺序操作。

## 一、解压数据集

当前数据在 `dataset/data/Benchmark/graspnet/` 下为 zip 文件，需全部解压到**同一目录**下。

```bash
cd /home/mu/graspnet-baseline/dataset/data/Benchmark/graspnet
unzip collision_label.zip
unzip dex_models.zip
unzip grasp_label.zip
unzip models.zip
unzip rect_labels.zip
unzip test_seen.zip
```

解压后，该目录下应包含至少：
- `grasp_label/`：物体抓取标签（000_labels.npz ~ 087_labels.npz）
- `collision_label/`：碰撞标签（按 scene 分目录）
- `scenes/`：场景数据（每个 scene 下有 realsense/kinect、rgb、depth、label、meta 等）

**说明**：若只有 `test_seen.zip`，解压后通常只有测试场景（如 scene_0100～scene_0129）。**训练**需要训练场景（scene_0000～scene_0099），需从 [GraspNet 官网](https://graspnet.net/) 下载完整数据集中的训练/场景部分并解压到同一 `graspnet` 目录。

---

## 二、生成 Tolerance 标签

原始数据不包含 tolerance 标签，训练前必须生成（或直接下载现成的）。

### 方式 A：本地生成（需已解压且包含 grasp_label）

```bash
cd /home/mu/graspnet-baseline/dataset
sh command_generate_tolerance_label.sh
```

脚本会从 `dataset/data/Benchmark/graspnet` 读取 `grasp_label`，在 `dataset/tolerance/` 下生成各物体的 `*_tolerance.npy`。  
可根据机器情况修改 `command_generate_tolerance_label.sh` 中的 `--num_workers`（例如 8 或 50）。

### 方式 B：直接下载（推荐，更快）

从 [Google Drive](https://drive.google.com/file/d/1DcjGGhZIJsxd61719N0iWA7L6vNEK0ci/view?usp=sharing) 或 [百度网盘](https://pan.baidu.com/s/1HN29P-csHavJF-R_wec6SQ) 下载 `tolerance.tar`，然后：

```bash
mv tolerance.tar /home/mu/graspnet-baseline/dataset/
cd /home/mu/graspnet-baseline/dataset
tar -xvf tolerance.tar
```

解压后应得到 `dataset/tolerance/` 目录及其中 `*_tolerance.npy` 文件。

---

## 三、训练

确认以下两点：
1. 数据集根目录已解压且包含 `scenes/`、`grasp_label/`、`collision_label/`（训练需 scene_0000～scene_0099）。
2. 已按「二」生成或下载并解压好 `dataset/tolerance/`。

修改并运行训练脚本（数据集路径请按实际修改）：

```bash
cd /home/mu/graspnet-baseline
# 编辑 command_train.sh，将 --dataset_root 改为你的 graspnet 根目录，例如：
# --dataset_root /home/mu/graspnet-baseline/dataset/data/Benchmark/graspnet
sh command_train.sh
```

训练日志和 checkpoint 会保存在 `--log_dir` 指定目录（如 `logs/log_rs`），可用 TensorBoard 查看。

---

## 四、测试与评估

需要已有 checkpoint（自己训练得到或下载预训练权重）。

- 预训练权重：  
  - [checkpoint-rs.tar](https://drive.google.com/file/d/1hd0G8LN6tRpi4742XOTEisbTXNZ-1jmk/view?usp=sharing)（RealSense）  
  - [checkpoint-kn.tar](https://drive.google.com/file/d/1vK-d0yxwyJwXHYWOtH1bDMoe--uZ2oLX/view?usp=sharing)（Kinect）

解压后，在 `command_test.sh` 中设置：
- `--dataset_root`：同上，指向解压后的 graspnet 根目录
- `--checkpoint_path`：解压得到的 checkpoint 文件（如 `checkpoint.tar`）
- `--dump_dir`：结果输出目录

然后执行：

```bash
sh command_test.sh
```

快速推理可将 `--collision_thresh` 设为 `-1`；复现论文评估结果时设为 `0.01`。

---

## 五、Demo（单张 RGB-D 推理）

使用预训练权重对单张 RGB-D 做抓取检测与可视化：

```bash
sh command_demo.sh
```

在 `command_demo.sh` 或 `demo.py` 中设置 `--checkpoint_path` 为你的 checkpoint 路径。  
使用自己的数据时，需在 `demo.py` 的 `get_and_process_data()` 中按 [doc/example_data/](doc/example_data/) 的格式准备 RGB-D 与相机内参。

## 路径汇总（按当前项目布局）

| 用途           | 路径 |
|----------------|------|
| 数据集根目录   | `/home/mu/graspnet-baseline/dataset/data/Benchmark/graspnet` |
| Tolerance 目录 | `/home/mu/graspnet-baseline/dataset/tolerance`（由脚本生成或 tar 解压） |
| 训练 / 测试时  | `--dataset_root` 填上述「数据集根目录」 |

如有报错，优先检查：  
1）zip 是否全部解压到同一 graspnet 目录；  
2）`scenes/`、`grasp_label/`、`collision_label/` 是否存在；  
3）训练前是否已有 `dataset/tolerance/`。

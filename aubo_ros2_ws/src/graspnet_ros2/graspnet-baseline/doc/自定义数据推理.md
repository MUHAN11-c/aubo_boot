# 使用自己的数据进行抓取推理

用 **demo** 对单张 RGB-D 图像做抓取检测与可视化。按下面准备一个数据目录，再指定 `--data_dir` 即可。

---

## 1. 数据目录里需要哪些文件

在**同一个文件夹**下放好这 4 个文件（名字固定）：

| 文件名 | 说明 |
|--------|------|
| **color.png** | RGB 彩色图，与深度图同分辨率 |
| **depth.png** | 深度图，与彩色图同分辨率；数值单位任意，用 `meta.mat` 里的 `factor_depth` 转成米 |
| **workspace_mask.png** | 工作空间 mask，与彩色/深度同分辨率；非零像素参与点云与抓取，可全白表示整幅图都有效 |
| **meta.mat** | MATLAB 格式，包含相机内参和深度缩放（见下） |

---

## 2. meta.mat 格式

用 Python 保存示例（也可用 MATLAB）：

```python
import scipy.io as scio
import numpy as np

# 相机内参 3x3：fx, fy 焦距，cx, cy 主点
# 若你已知 fx, fy, cx, cy：
intrinsic_matrix = np.array([
    [fx,  0, cx],
    [ 0, fy, cy],
    [ 0,  0,  1]
])
# 深度图数值转成米的系数：若深度图单位是毫米，则 factor_depth = 1000
factor_depth = 1000.0   # 或 1.0（若已是米）

scio.savemat('meta.mat', {
    'intrinsic_matrix': intrinsic_matrix,
    'factor_depth': np.array([factor_depth])  # 标量也建议包成数组
})
```

- **intrinsic_matrix**：3×3，`[fx, 0, cx; 0, fy, cy; 0, 0, 1]`，与 **color/depth 分辨率一致**（即该内参是针对当前图像尺寸的）。
- **factor_depth**：标量。深度图像素值 ÷ factor_depth = 米（例如深度是毫米则 1000）。

---

## 3. 运行命令

在项目根目录执行：

```bash
# 使用你的数据目录
python demo.py --checkpoint_path logs/log_kn/checkpoint-rs.tar --data_dir /path/to/your/data

# 或用脚本（在 command_demo.sh 里加上 --data_dir）
sh command_demo.sh
```

若用脚本，可在 `command_demo.sh` 里加上一行（或把默认 `data_dir` 改成你的路径）：

```bash
CUDA_VISIBLE_DEVICES=0 python demo.py --checkpoint_path logs/log_kn/checkpoint-rs.tar --data_dir /path/to/your/data
```

运行后会弹出 Open3D 窗口，显示点云和 Top-50 抓取；关闭窗口即结束。

---

## 4. 可选参数

- `--num_point 20000`：从点云中采样的点数，默认 20000。
- `--collision_thresh 0.01`：碰撞检测阈值；设为 `-1` 不做碰撞检测，只做检测+可视化。
- `--voxel_size 0.01`：碰撞检测前体素大小（米）。

---

## 5. 分辨率说明

- **color.png** 和 **depth.png** 必须**同分辨率**。
- **meta.mat** 里的 **intrinsic_matrix** 必须是对**该分辨率**标定的内参（fx, fy, cx, cy 与宽高一致）。
- 程序会根据你的图像尺寸自动用对应宽高，无需固定 1280×720。

---

## 6. 没有 workspace_mask 时

若暂时没有 mask，可以：

- 用同尺寸的全白图（255）作为 `workspace_mask.png`，表示整幅图都参与；
- 或用图像处理生成简单 mask（例如只保留桌面区域）。

参考示例数据：`doc/example_data/`。

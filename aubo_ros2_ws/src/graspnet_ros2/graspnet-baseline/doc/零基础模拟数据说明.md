# 零基础：模拟数据是如何一步一步实现的

demo 的 `get_and_process_data(data_dir)` 需要从 `data_dir` 里读到 4 样东西。下面用「零基础」的方式说明：每一样是什么、在代码里怎么生成、和后续处理怎么对应。

---

## 1. 先约定「图像尺寸」

- 所有数据都基于**同一张“虚拟相机”拍出来的图**。
- 这张图有 **高（行数）** 和 **宽（列数）**，例如 **480×640**。
- **color、depth、workspace_mask 三张图必须是同一尺寸**（都是 480×640），否则后面按像素对齐会错乱。

**对应脚本**：`create_fake_data_for_demo.py` 里的 `HEIGHT = 480`、`WIDTH = 640`。

---

## 2. 第一样：color.png（彩色图）

**是什么**：一张 RGB 图，每个像素 3 个数 (R, G, B)，范围 0~255。

**在 demo 里怎么用**：读进来后除以 255 变成 0~1 的 float，和点云**按像素一一对应**，用来给 3D 点“上色”（可视化和部分模型会用到）。

**模拟时怎么做**：
1. 开一个大小为 `(高, 宽, 3)` 的数组，类型 `uint8`。
2. 按你的想法给不同区域填上 R、G、B（例如上半蓝、下半绿、中间一块灰当“假物体”）。
3. 用 `PIL.Image.fromarray(...).save('color.png')` 存成 PNG。

**和 get_and_process_data 的对应**：  
`color = np.array(Image.open(...'color.png'), dtype=np.float32) / 255.0`，后面用 `mask` 筛出有效点后得到 `color_masked`，再采样得到 `color_sampled`，放进 `end_points['cloud_colors']`。

---

## 3. 第二样：depth.png（深度图）

**是什么**：和彩色图**同尺寸**的一张图，每个像素**一个数**，表示“这个像素对应的 3D 点离相机有多远”。  
在 GraspNet 里通常不直接存“米”，而是存**原始深度值**，满足：

- **真实深度（米）= 深度图像素值 / factor_depth**

所以你要和后面的 `meta.mat` 里 `factor_depth` 一起约定好。

**在 demo 里怎么用**：  
用深度图 + 相机内参做**反投影**，得到每个像素对应的 3D 点 (x, y, z)，单位米，即有序点云 `cloud`。

**模拟时怎么做**：
1. 先按“米”想好每个像素的深度，例如背景 0.8 米、中间“物体”0.4 米，得到一个 `(高, 宽)` 的数组 `depth_meters`。
2. 设一个 **factor_depth**（例如 1000），算 **深度图要存的值 = depth_meters * factor_depth**。
3. 把结果转成 0~65535 的整数（例如 `uint16`），保存成 `depth.png`。  
这样读进来后 `depth像素值 / factor_depth` 就得到米。

**和 get_and_process_data 的对应**：  
- `depth = np.array(Image.open(...'depth.png'))`  
- `mask = (workspace_mask & (depth > 0))`：深度为 0 的像素不参与（无效深度）。  
- `create_point_cloud_from_depth_image(depth, camera, organized=True)` 里会用 `depth/camera.scale`（即 `depth/factor_depth`）得到 Z（米），再结合内参算 X、Y。

---

## 4. 第三样：workspace_mask.png（工作空间掩码）

**是什么**：和彩色/深度**同尺寸**的一张图，每个像素一个数。**非零**表示“这个像素在工作空间内，要参与抓取”；0 表示忽略。

**在 demo 里怎么用**：  
`mask = (workspace_mask & (depth > 0))`，只保留“在工作空间内且深度有效”的像素，再用这个 `mask` 从 `cloud` 和 `color` 里取出有效点 `cloud_masked`、`color_masked`。

**模拟时怎么做**：
1. 开一个 `(高, 宽)` 的数组，初始全 0。
2. 把“工作空间”区域（例如图像中间一个矩形）设为 255（或 1）。
3. 存成 `workspace_mask.png`。

**和 get_and_process_data 的对应**：  
`workspace_mask = np.array(Image.open(...'workspace_mask.png'))`，然后和 `depth > 0` 一起得到 `mask`，再 `cloud_masked = cloud[mask]`。

---

## 5. 第四样：meta.mat（相机参数）

**是什么**：一个 MATLAB 格式文件，里面至少要有：
- **intrinsic_matrix**：3×3 相机内参矩阵，形式一般是  
  `[ fx  0  cx ]`  
  `[ 0  fy  cy ]`  
  `[ 0  0   1 ]`  
  - fx, fy：焦距（像素单位）  
  - cx, cy：主点（通常接近图像中心 宽/2、高/2）
- **factor_depth**：深度缩放因子，满足 **真实深度(米) = depth 图像素值 / factor_depth**。

**在 demo 里怎么用**：  
用 `intrinsic` 的 fx, fy, cx, cy 和 `factor_depth` 构造 `CameraInfo`，再调用 `create_point_cloud_from_depth_image(depth, camera, organized=True)` 把深度图反投影成 3D 点云。

**模拟时怎么做**：
1. 选一个合理的 fx、fy（例如 600），cx = 宽/2，cy = 高/2，组成 3×3 的 `intrinsic_matrix`。
2. `factor_depth` 必须和生成 depth 时用的**完全一致**（例如 1000）。
3. 用 `scipy.io.savemat('meta.mat', {'intrinsic_matrix': ..., 'factor_depth': ...})` 保存。

**和 get_and_process_data 的对应**：  
`meta = scio.loadmat(...'meta.mat')`，`intrinsic = meta['intrinsic_matrix']`，`factor_depth = meta['factor_depth']`，然后传给 `CameraInfo` 和反投影。

---

## 6. 整体流程对应（从模拟到 demo）

| 步骤 | 模拟数据脚本里做的 | get_and_process_data 里用的 |
|------|---------------------|-----------------------------|
| 1 | 定 HEIGHT, WIDTH | 从 depth.shape 得到 height, width |
| 2 | 生成并保存 color.png | Image.open → 归一化 → color |
| 3 | 生成 depth_meters，乘 factor_depth 存 depth.png | Image.open → depth；mask 里 depth>0 |
| 4 | 生成并保存 workspace_mask.png | Image.open → workspace_mask；mask 里 workspace_mask 非零 |
| 5 | 用 fx,fy,cx,cy,factor_depth 存 meta.mat | loadmat → CameraInfo → 反投影 |
| 6 | （自检）用同一公式把 depth 变点云 | create_point_cloud_from_depth_image |
| 7 | — | mask 筛出 cloud_masked / color_masked |
| 8 | — | 采样到 num_point → end_points + Open3D cloud |

---

## 7. 如何运行

在 graspnet-baseline 目录下：

```bash
# 生成模拟数据到 doc/fake_pose_1
python create_fake_data_for_demo.py --out_dir doc/fake_pose_1

# 用生成的数据跑 demo（需已有 checkpoint）
python demo.py --data_dir doc/fake_pose_1 --checkpoint_path <你的权重路径>
```

这样就从零开始，用“假”的 color、depth、workspace_mask、meta 完整走通 demo 的数据管线。

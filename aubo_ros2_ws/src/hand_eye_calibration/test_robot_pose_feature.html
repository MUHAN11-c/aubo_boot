🧪 测试步骤

前置条件

  1. 启动手眼标定节点：ros2 launch hand_eye_calibration hand_eye_calibration_launch.py
  2. 确保机器人状态话题 /robot_status 正常发布
  3. 打开浏览器访问：http://localhost:8080
  4. 切换到"手眼标定精度验证"选项卡

功能测试

  1. 点击"获取机器人测试姿态"按钮
  2. 检查测试高度编辑框：
    • 应该自动填充机器人当前Z轴值（毫米）
    • 例如：如果机器人Z=200.5mm，则显示200
  3. 检查平移高度编辑框：
    • 应该自动填充机器人Z轴值+20（毫米）
    • 例如：如果机器人Z=200.5mm，则显示220
  4. 检查验证结果数据右侧：
    • 应该显示"🤖 当前机器人姿态"区域
    • 格式：X: 123.4 mm; Y: 456.7 mm; Z: 200.5 mm; Qx: 0.000; Qy: 0.000; Qz: 0.000; Qw: 1.000
  5. 检查日志区域：
    • 应该显示"机器人姿态获取成功，高度参数已自动设置"
    • 应该显示详细的参数设置信息

🔧 技术实现细节

API调用

  • 接口：GET /api/robot_status
  • 返回数据：包含机器人完整状态和位姿信息
  • 单位处理：机器人位置单位是米，显示时转换为毫米

DOM操作

  • 测试高度：document.getElementById('min-test-height')
  • 平移高度：document.getElementById('translation-height')
  • 姿态显示：pose-x-display 到 pose-qw-display

样式设计

  • 布局：CSS Grid 2列布局，响应式调整为1列
  • 颜色：渐变背景，蓝色主题色
  • 字体：等宽字体显示数值，便于阅读

📝 修改的文件

  • index.html：添加机器人姿态显示区域
  • script_v2.js：完善getRobotTestPose函数，添加updateCurrentRobotPoseDisplay函数
  • style.css：添加机器人姿态显示样式

🎯 使用场景

这个功能特别适用于：

  • 手眼标定精度验证：快速获取当前机器人位置作为测试基准
  • 高度参数设置：自动计算合适的测试高度和平移高度
  • 姿态监控：实时查看机器人的完整姿态信息
  • 参数记录：为后续的精度测试提供参考数据

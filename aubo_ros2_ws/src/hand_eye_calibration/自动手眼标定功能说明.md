# 自动手眼标定功能说明

## 📋 概述

在现有的手眼标定系统中添加了"🤖 自动手眼标定"选项卡，与"相机标定精度验证"、"手眼标定"、"手眼标定精度验证"三个选项卡并列。

**添加时间**: 2025-01-04  
**版本**: v1.0

---

## 🎯 功能特点

### 1. **全自动流程**
- 自动规划采样位姿（均匀网格分布）
- 自动控制机器人移动到目标位姿
- 自动采集图像和提取棋盘格角点
- 自动执行手眼标定计算
- 无需人工干预，一键完成

### 2. **灵活的参数设置**
- **采样点数**: 8-20个（默认12个）
- **工作空间范围**: 100-500mm（默认300mm）
- **Z轴高度**: 200-500mm（默认300mm）
- **标定类型**: 眼在手外/眼在手上
- **棋盘格大小**: 可调整（默认15mm）

### 3. **实时进度监控**
- 进度条显示（百分比和数量）
- 当前状态显示（运行中/已暂停/已完成等）
- 当前步骤详情（移动中/采集图像/提取角点等）
- 实时操作日志记录

### 4. **灵活控制**
- **开始**: 启动自动标定流程
- **暂停**: 暂停当前流程，可继续
- **继续**: 从暂停处继续执行
- **停止**: 终止流程，保留已采集数据

### 5. **数据管理**
- 自动更新数据表格（实时显示）
- 支持保存位姿数据到JSON文件
- 支持加载已保存的位姿数据
- 支持手动触发标定计算

## ⚙️ 技术实现

### 位姿规划算法

采用**均匀网格分布**算法：

```javascript
// 计算网格大小
gridSize = ceil(sqrt(sampleCount))

// 计算步长
step = workspaceRange / (gridSize - 1)

// 生成网格点
for i in 0..gridSize:
    for j in 0..gridSize:
        x = -offset + i * step
        y = -offset + j * step
        z = zHeight (固定)
```

**示例**: 12个采样点
- 网格: 4×4 = 16个点（取前12个）
- 范围: ±150mm（总300mm）
- 步长: 100mm
- 分布: 均匀覆盖工作空间

### 数据流程

```
开始自动标定
    ↓
规划N个位姿 → [pose1, pose2, ..., poseN]
    ↓
━━━━━━━━━━━━━━━━━━━━━━━━━━━
循环 for i in 1..N:
    ↓
    1. 移动机器人到 pose[i]
    ↓
    2. 等待稳定 (1秒)
    ↓
    3. 采集图像
    ↓
    4. 提取棋盘格角点
    ↓
    5. 保存数据 {pose, corners}
    ↓
    6. 更新进度和表格
━━━━━━━━━━━━━━━━━━━━━━━━━━━
    ↓
检查数据量 >= 3?
    ↓ 是
自动执行标定计算
    ↓
显示标定结果（误差、矩阵）
    ↓
完成
```

### API接口

自动标定使用以下现有API：

1. **图像采集**: `GET /api/camera/capture`
2. **角点提取**: `POST /api/camera/extract_corners`
3. **手眼标定**: `POST /api/hand_eye/calibrate`
4. **机器人位姿**: `GET /api/robot/pose`
5. **机器人移动**: `POST /api/robot/move_to_pose` (待实现)

- 继续时恢复执行

- 采集的数据存储在 `autoCalibState.collectedData`
- 支持保存到JSON文件
- 支持从JSON文件加载
- 格式: `[{group_id, pose, corners, image_with_corners}, ...]`

## 🐛 已知问题与修复

### 已修复的问题

1. **JSON解析错误** (2025-01-04 17:45修复)
   - **问题**: `Unexpected token '<'` 错误
   - **原因**: fetch请求未检查响应状态，服务器返回HTML错误页面时尝试解析JSON
   - **修复**: 
     ```javascript
     // 添加响应状态检查
     if (!response.ok) {
         throw new Error(`HTTP ${response.status}: ${response.statusText}`);
     }
     
     // 检查content-type
     const contentType = response.headers.get('content-type');
     if (!contentType || !contentType.includes('application/json')) {
         const text = await response.text();
         throw new Error(`服务器返回非JSON响应`);
     }
     ```
   - **影响**: 所有fetch调用（图像采集、角点提取、标定计算）

2. **HTTP方法错误** (2025-01-04 17:47修复)
   - **问题**: `HTTP 405: METHOD NOT ALLOWED` 错误
   - **原因**: `/api/camera/capture` 端点定义为POST方法，但前端使用了默认的GET方法
   - **修复**:
     ```javascript
     // 修改前（错误）
     const response = await fetch('/api/camera/capture');
     
     // 修改后（正确）
     const response = await fetch('/api/camera/capture', {
         method: 'POST',
         headers: {'Content-Type': 'application/json'}
     });
     ```
   - **影响**: 手动采集图像和自动标定流程中的图像采集

3. **图像不显示/placeholder不隐藏** (2025-01-04 17:50修复)
   - **问题**: 图像采集成功后，右侧图像区域不显示图像或显示被placeholder遮挡
   - **原因**: 缺少图像加载完成后的处理逻辑，未隐藏placeholder
   - **修复**:
     ```javascript
     // 添加图像加载处理
     imgElement.onload = function() {
         this.classList.add('loaded');
         if (placeholder) {
             placeholder.style.display = 'none';
         }
     };
     // 添加时间戳避免缓存
     imgElement.src = data.image_data + '?t=' + new Date().getTime();
     ```
   - **影响**: 所有图像显示（采集图像、提取角点、自动流程）
   - **效果**: 图像加载后自动隐藏"等待图像..."提示，显示清晰图像

## 🔧 常见问题排查

### 1. "Unexpected token '<'" 错误

**错误信息**:
```
❌ 请求失败：Unexpected token '<', "<!DOCTYPE..."
```

**原因**: 
- API端点不存在或路径错误
- 服务器返回了HTML错误页面而不是JSON
- Flask应用未正常运行

**解决方法**:
1. 检查Flask服务是否正常运行
   ```bash
   # 查看启动脚本输出
   tail -f /home/mu/IVG/aubo_ros2_ws/src/hand_eye_calibration/start_hand_eye_calibration.sh
   ```

2. 检查浏览器控制台网络请求
   - 打开浏览器开发者工具（F12）
   - 查看Network标签
   - 查看失败请求的Response内容

3. 验证API端点
   ```bash
   # 测试API是否可访问
   curl http://localhost:5000/api/camera/capture
   ```

4. 检查日志输出
   - 查看ROS2节点日志
   - 查看Flask应用日志

### 2. HTTP 405 方法不允许

**错误信息**:
```
❌ 请求失败：HTTP 405: METHOD NOT ALLOWED
```

**原因**: 
- 使用了错误的HTTP方法（GET/POST不匹配）

**解决方法**:
- 此问题已在v1.0.2版本修复
- 如果仍遇到，请检查是否使用了旧版本的JavaScript文件
- 强制刷新浏览器（Ctrl+F5）清除缓存

### 3. 图像采集失败

**错误信息**:
```
❌ 图像采集失败：HTTP 500
```

**可能原因**:
- 相机未连接或离线
- ROS2话题无数据
- 相机驱动未启动

**解决方法**:
```bash
# 检查相机话题
ros2 topic list | grep camera

# 查看话题数据
ros2 topic echo /camera/color/image_raw --once

# 重启相机节点
ros2 launch percipio_camera_interface camera.launch.py
```

### 4. 角点提取失败

**错误信息**:
```
❌ 角点提取失败：未检测到棋盘格
```

**可能原因**:
- 光照不足或过亮
- 棋盘格位置不正确
- 棋盘格尺寸参数设置错误
- 图像模糊

**解决方法**:
1. 调整棋盘格位置和角度
2. 改善光照条件
3. 验证棋盘格参数（列数×行数，格子大小）
4. 确保相机对焦清晰

### 5. 自动标定无法启动

**错误信息**:
```
⚠️ 请先加载相机参数
```

**解决方法**:
1. 等待2-3秒让系统从ROS2话题获取相机参数
2. 查看左下角"相机标定参数"是否显示内参
3. 手动点击"加载相机参数"按钮
4. 检查ROS2 CameraInfo话题是否发布
   ```bash
   ros2 topic echo /camera/color/camera_info --once
   ```

### 6. 机器人未响应

**错误信息**:
```
机器人状态: 离线
```

**解决方法**:
1. 检查机器人是否上电
2. 检查网络连接
3. 验证ROS2 robot_status话题
   ```bash
   ros2 topic list | grep robot
   ros2 topic echo robot_status --once
   ```

### 7. 标定误差过大

**表现**:
```
📊 标定误差：15.234 mm
```

**可能原因**:
- 采样点数量不足
- 位姿分布不均匀
- 角点提取不准确
- 标定板测量误差

**优化方法**:
1. 增加采样点数量（12-15个）
2. 扩大工作空间范围
3. 确保棋盘格平整
4. 提高光照质量
5. 重新测量棋盘格实际尺寸

## 检验结果

### ✅ 1. 像素到相机坐标转换

**公式**：
```
X = (u - cx) * Z / fx
Y = (v - cy) * Z / fy
Z = Z
```

**测试数据**：
- 相机内参：fx=464.95, fy=465.01, cx=326.17, cy=240.68
- 测试像素：(326.0, 240.0)（接近图像中心）
- 深度：500.0 mm

**计算结果**：
- X = -0.183 mm
- Y = -0.731 mm
- Z = 500.000 mm

**验证**：✅ 正确（中心点应接近 (0, 0, Z)）

---

### ✅ 2. 相邻角点距离计算

**方法**：
- 水平相邻：当前点到右侧点的距离
- 垂直相邻：当前点到下侧点的距离

**测试数据**：
- 3x3 棋盘格（3列3行，9个角点）
- 格子大小：15.0 mm
- 理想3D点（Z=500mm平面）

**计算结果**：
- 水平相邻距离：6个，全部为 15.000 mm
- 垂直相邻距离：6个，全部为 15.000 mm
- 平均距离：15.000 mm
- 误差：0.000000 mm

**验证**：✅ 正确

---

### ✅ 3. 对角线距离计算

**公式**：
```
理论对角线距离 = sqrt((cols-1)² + (rows-1)²) * square_size
```

**测试数据**：
- 10列 x 7行 棋盘格（70个角点）
- 格子大小：15.0 mm
- 对角线跨越：9列 x 6行

**计算过程**：
```
sqrt(9² + 6²) * 15.0
= sqrt(81 + 36) * 15.0
= sqrt(117) * 15.0
= 10.816654 * 15.0
= 162.250 mm
```

**验证**：
- 第一个角点：(0, 0, 500)
- 最后一个角点：(135, 90, 500)
- 实际对角线距离：162.250 mm
- 误差：0.000000 mm

**验证**：✅ 正确

---

### ✅ 4. 深度缩放因子

**公式**（与 depth_z_reader 一致）：
```
depth_meters = depth_raw * 0.00025
depth_mm = depth_meters * 1000
```

**测试数据**：
- 深度原始值：2000
- 缩放因子：0.00025

**计算结果**：
- 深度（米）：2000 * 0.00025 = 0.5 m
- 深度（毫米）：0.5 * 1000 = 500.0 mm

**验证**：✅ 正确（与 depth_z_reader 完全一致）

## 误差来源分析

**0.71%的误差（优秀）来自**：

1. **相机标定精度**（主要）
   - 内参矩阵的标定误差
   - 畸变系数的拟合误差

2. **深度测量精度**
   - 深度相机的测量精度（通常±1-2mm）
   - 深度图的量化误差（16位）

3. **物理因素**
   - 棋盘格实际加工精度（可能±0.1mm）
   - 温度变化导致的材料膨胀/收缩

4. **图像处理精度**
   - 角点检测的亚像素精度（通常±0.1像素）
   - 畸变校正的插值误差

## 发现并修复的问题

### `calculate_adjacent_distances` 函数

**问题**：
- 最后一列的点使用 `distances[-1]` 作为占位符
- 这是不正确的逻辑

**影响**：
- 仅影响显示值，不影响统计计算（因为 `calculate_max_distance_error` 独立计算）

**修复**：
- 改为计算下侧相邻距离
- 右下角点使用 0.0

## 基本使用流程

### 示例 1: 检测单个工件

**场景**: 检测桌面上的齿轮

**步骤**:

1. **准备图像**
   - 使用相机拍摄包含齿轮的图像
   - 确保光照均匀，背景简单
   - 保存为 JPG 或 PNG 格式

2. **上传图像**
   ```
   点击上传区域 → 选择图像文件 → 等待预览显示
   ```

3. **设置参数**
   ```
   工件ID: gear_01
   ```

4. **执行检测**
   ```
   点击 "估计姿态" 按钮 → 等待处理（约3-5秒）
   ```

5. **查看结果**
   - 检测数量: 1
   - 平均置信度: 95.3%
   - 可视化图像: 显示检测框和姿态
   - 位置坐标: (X: 0.25, Y: 0.13, Z: 0.45)

### 示例 2: 批量检测多个工件

**场景**: 检测装配线上的多个螺栓

**步骤**:

1. **上传包含多个目标的图像**
   - 图像中包含3-5个螺栓
   - 确保各个螺栓清晰可见

2. **设置参数**
   ```
   工件ID: bolt_m8
   ```

3. **执行检测**
   - 系统会自动检测图像中的所有匹配目标

4. **分析结果**
   ```
   检测数量: 4
   平均置信度: 92.7%
   
   对象 1: 置信度 95.2%, 位置 (0.10, 0.20, 0.40)
   对象 2: 置信度 93.8%, 位置 (0.15, 0.25, 0.42)
   对象 3: 置信度 91.5%, 位置 (0.20, 0.30, 0.41)
   对象 4: 置信度 90.3%, 位置 (0.25, 0.35, 0.43)
   ```

## 高级功能

### 功能 1: 模板管理

**创建新模板**:

1. 准备标准姿态的工件图像
2. 使用姿态估计功能验证检测效果
3. 如果效果不佳，需要添加更多训练样本

**模板标准化**:

1. 选择要标准化的工件
   ```
   在模板列表中点击目标工件
   或在工件ID框中输入: gear_01
   ```

2. 点击"标准化模板"按钮
   ```
   处理时间: 约30-120秒（取决于模板数量）
   ```

3. 查看结果
   ```
   处理完成！
   处理: 5个姿态
   跳过: 1个姿态（已存在标准化版本）
   ```

### 功能 2: 结果导出

虽然当前版本暂不支持一键导出，但你可以：

1. **截图保存结果**
   - 使用浏览器截图工具
   - 保存可视化图像

2. **记录坐标数据**
   - 手动复制位置坐标
   - 记录置信度信息

3. **查看ROS2日志**
   ```bash
   ros2 topic echo /pose_estimation_result
   ```

## 批量处理

### 场景: 处理一批工件图像

**方法 1: 手动逐个处理**

```
对于每张图像:
  1. 上传图像
  2. 输入工件ID
  3. 点击估计姿态
  4. 记录结果
  5. 清除并继续下一张
```

**方法 2: 使用ROS2命令行**

如需批量处理，建议使用ROS2服务直接调用：

```bash
# 创建批处理脚本
for image in *.jpg; do
  ros2 service call /estimate_pose interface/srv/EstimatePose \
    "{image: $(cat $image | base64), object_id: 'workpiece_01'}"
done
```

## 最佳实践

### 1. 图像采集建议

✅ **推荐做法**:
- 使用固定光源，避免阴影
- 保持相机与工件的距离一致
- 确保工件完整出现在图像中
- 背景尽量简单、单色

❌ **避免**:
- 过曝或欠曝的图像
- 模糊或失焦的图像
- 遮挡严重的工件
- 复杂背景干扰

### 2. 工件ID命名规范

**建议格式**:
```
[类型]_[规格]_[版本]

示例:
- gear_20t_v1    (齿轮, 20齿, 版本1)
- bolt_m8_v2     (螺栓, M8规格, 版本2)
- plate_100x50   (板材, 100x50mm)
```

### 3. 模板管理策略

**何时创建新模板**:
- 新增工件类型
- 现有模板检测效果不佳
- 工件外观发生变化

**何时标准化模板**:
- 添加新的姿态样本后
- 模板目录结构调整后
- 定期维护（建议每月一次）

### 4. 性能优化

**加快检测速度**:
- 使用适当分辨率的图像（推荐: 1920x1080）
- 避免过大的图像文件（<5MB）
- 确保模板数量适中（每个工件5-10个姿态）

**提高检测精度**:
- 增加不同角度的模板样本
- 使用高质量的训练图像
- 定期更新和维护模板

## 常见场景问题解决

### 问题 1: 检测不到目标

**可能原因**:
- 模板不匹配
- 图像质量差
- 光照条件变化

**解决方法**:
1. 检查工件ID是否正确
2. 尝试上传不同角度的图像
3. 调整相机参数或光照
4. 添加更多模板样本

### 问题 2: 置信度过低

**可能原因**:
- 遮挡严重
- 背景干扰
- 模板质量不佳

**解决方法**:
1. 改善拍摄环境
2. 使用更清晰的图像
3. 重新训练模板
4. 增加训练样本数量

### 问题 3: 检测速度慢

**可能原因**:
- 图像尺寸过大
- 模板数量过多
- 系统资源不足

**解决方法**:
1. 压缩图像到合适大小
2. 精简模板库
3. 优化系统配置
4. 考虑使用GPU加速

# 快速开始指南

## 🚀 5分钟快速上手

### 前提条件

✅ ROS2环境已配置  
✅ visual_pose_estimation_python 已编译  
✅ Python3 环境可用

### 步骤1: 启动ROS2节点

打开终端1，启动姿态估计服务：

```bash
# source ROS2环境
source /opt/ros/humble/setup.bash

# source工作空间
cd /home/mu/IVG/aubo_ros2_ws
source install/setup.bash

# 启动姿态估计节点
ros2 launch visual_pose_estimation_python visual_pose_estimation_python.launch.py
```

### 步骤2: 启动Web UI

打开终端2，启动Web界面：

```bash
cd /home/mu/IVG/aubo_ros2_ws/src/visual_pose_estimation/src/visual_pose_estimation_python/web_ui

# 启动Web UI
./start_web_ui.sh
```

看到以下信息表示启动成功：

```
========================================
✓ Web UI 启动成功！
========================================

访问地址:
  http://localhost:8089
  http://localhost:8089/index.html
```

### 步骤3: 打开浏览器

在浏览器中访问：

```
http://localhost:8089
```

### 步骤4: 使用界面

1. **上传图像**
   - 点击上传区域
   - 选择一张包含目标物体的图像

2. **输入工件ID**
   - 在"工件ID"框中输入，例如: `workpiece_01`

3. **开始估计**
   - 点击"估计姿态"按钮
   - 等待几秒钟
   - 查看右侧结果

4. **管理模板**
   - 滚动到底部"模板管理"区域
   - 点击"刷新模板列表"
   - 浏览和选择模板

### 停止服务

在终端2中按 `Ctrl+C` 或运行：

```bash
./stop_web_ui.sh
```

## 🎯 示例工作流

### 场景: 检测桌面上的工件

1. 准备一张包含工件的图像
2. 确保对应的模板已创建在 templates 目录
3. 上传图像
4. 输入工件ID (例如: `gear_01`)
5. 点击"估计姿态"
6. 查看检测结果：
   - 检测数量
   - 置信度
   - 可视化图像
   - 各个检测对象的详细位姿

## 📝 常见问题

### Q: 启动失败，提示端口被占用？

**A**: 运行停止脚本清理：
```bash
./stop_web_ui.sh
```

### Q: 点击"估计姿态"后长时间无响应？

**A**: 检查：
1. ROS2节点是否在运行
2. 终端是否有错误日志
3. 模板文件是否存在

### Q: 模板列表为空？

**A**: 确认：
1. templates目录是否存在
2. 目录中是否有模板文件
3. 模板文件格式是否正确

### Q: 图像上传失败？

**A**: 
1. 确保图像格式为 JPG 或 PNG
2. 图像大小建议小于 10MB
3. 检查浏览器控制台是否有错误

## 🔍 调试技巧

### 查看ROS2服务

```bash
# 列出所有服务
ros2 service list

# 查看服务详情
ros2 service type /estimate_pose
```

### 手动测试服务

```bash
# 测试estimate_pose服务
ros2 service call /estimate_pose interface/srv/EstimatePose "{object_id: 'workpiece_01'}"

# 测试list_templates服务
ros2 service call /list_templates interface/srv/ListTemplates
```

### 查看Web服务状态

在浏览器中访问：
```
http://localhost:8089/status
```

应该返回：
```json
{
  "status": "online",
  "port": 8089,
  "service": "visual_pose_estimation_python_http_bridge",
  "timestamp": "..."
}
```
